# Product Overview

## Problem
Experienced candidates with hybrid backgrounds often waste time on strong-fit roles because the market sorts them poorly. The product needs to reduce ambiguity, improve role selection, sharpen positioning, and create reusable application assets.

## Primary user
Single-user expert mode.
The first user is a senior communications and marketing professional targeting high-fit roles in regulated sectors.

## Jobs to be done
1. Assess whether a job is worth pursuing.
2. Decide which professional lane to use.
3. Retrieve the best proof points from prior experience.
4. Generate tailored assets quickly.
5. Track responses and learn from outcomes.

## In scope
- Manual job ingestion
- Job parsing and lane classification
- Fit scoring
- Achievement retrieval
- Resume summary and bullet generation
- Cover letter generation
- Networking and recruiter message drafts
- CRM tracking for jobs, contacts, and outcomes
- Weekly strategy summaries

## Out of scope for v1
- Browser automation for applications
- LinkedIn scraping
- Fully autonomous outreach
- Multi-user collaboration
- Native mobile app
- Voice interface

## Success metrics
- Time from job intake to first draft under 10 minutes
- Manual review completion rate over 90%
- Application generation quality rated acceptable by user in at least 80% of cases
- Ability to identify top converting lane within 30 tracked applications

## Core user flows
### Flow 1: Intake and decision
Paste job description -> parse -> score -> route lane -> approve or discard.

### Flow 2: Application build
Approved job -> retrieve best proof points -> generate summary, bullets, cover letter, outreach -> review -> save.

### Flow 3: Strategy loop
Update outcome -> aggregate analytics -> weekly recommendation.

## Product philosophy
The system should optimize for:
- clarity over cleverness
- retrieval over long-memory prompting
- structured outputs over freeform generation
- human approval over automation theater
